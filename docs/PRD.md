## 1. 产品定位

### 1.1 一句话定义

**栖知是一个围绕用户个人资料、学习目标、知识图谱与学习记录构建的 AI 私人学习空间。**

它不是普通聊天机器人，也不是单纯资料管理器，而是一个能把“资料 → 理解 → 练习 → 笔记 → 路径推进”串起来的桌面学习工作台。

### 1.2 核心价值

1. **可信学习**：回答优先基于用户资料，但不能把弱相关资料伪装成证据。
2. **干净上下文**：每个学习会话必须清楚知道自己带了什么上下文；新建会话默认必须为空白。
3. **学习闭环**：资料、目标、图谱、笔记、练习、番茄钟不能互相孤立。
4. **低打扰自动化**：AI 可以自动开面板、写笔记、启动番茄钟，但必须可关闭、可解释。
5. **长期个性化**：学习画像与学习记录逐步影响讲解方式、题目难度和复习提醒。

## 2. 用户与场景

### 2.1 目标用户

首发用户：大学生、考研/考证用户、自主学习编程或专业课的人。

典型特征：
- 有自己的 PDF、课件、笔记、题库。
- 学习内容杂，容易断上下文。
- 希望 AI 帮忙解释、总结、出题，但不接受胡说。
- 需要阶段目标、复习提醒和进度反馈。

### 2.2 核心场景

| 场景 | 用户动作 | 栖知响应 |
|---|---|---|
| 自由提问 | 在学习空间直接问问题 | 判断是否学习问题；学习问题可检索资料，闲聊/身份问题不触发 RAG |
| 带资料学习 | 从资料库进入学习空间 | 带入 resourceId/nodeId，回答优先使用当前资料 |
| 带目标学习 | 从目标任务进入学习空间 | 带入 taskId，学习完成后可更新任务状态 |
| 带笔记复习 | 从笔记进入学习空间 | 带入 noteId，围绕笔记继续问答或整理 |
| 图谱推进 | 从知识图谱节点进入学习空间 | 带入 nodeId，展示当前位置和前后依赖 |
| 练习巩固 | 点击生成练习 | 只在强证据 RAG 命中时生成题，并保留证据回链 |

## 3. 产品信息架构

```text
栖知
├─ 看板 Dashboard：今日学习概览、进度、提醒
├─ 学习空间 Study：核心问答与学习执行区
├─ 我的目标 Goals：学习目标、阶段、任务
├─ 资料库 Library：上传、解析、摘要、检索来源
├─ 知识图谱 Graph：知识节点、依赖关系、路径推进
├─ 笔记 Notes：用户笔记、AI 整理、复习入口
└─ 设置 Settings：模型、自动化、记忆与数据
```

### 3.1 Study 是执行中心，不是所有状态的来源

Study 负责“当前学习会话”的呈现与交互，但资料、目标、笔记、图谱的主数据分别由各自模块维护。

### 3.2 其他模块进入 Study 时只传上下文

允许传入：
- `source`: `library | graph | goal | note | free`
- `resourceId`
- `nodeId`
- `taskId`
- `noteId`

Study 必须根据这些上下文决定是否自动展示资源/图谱/笔记面板。

## 4. 学习空间产品规则

### 4.1 新建会话规则

新建自由学习会话必须满足：
- 不自动插入任何 assistant 消息。
- 不自动打开图谱/资料/笔记/番茄钟面板。
- 标题初始为“新的学习会话”或“自由学习会话”。
- 用户发送第一条消息后再生成会话标题。
- 会话标题不得包含 `<think>`、模型分析或系统提示。

### 4.2 意图识别规则

进入回答前先判断意图：

| 意图 | 示例 | 行为 |
|---|---|---|
| 身份/闲聊 | 你是谁、你能做什么、你好 | 不触发 RAG，不展示证据卡 |
| 学习解释 | 解释中值定理、什么是闭包 | 可触发 RAG |
| 资料追问 | 根据这份资料讲一下 | 必须优先当前资料 |
| 练习/测验 | 给我出题、我懂了 | 进入评估/路径推进 |
| 工具操作 | 开番茄钟、整理笔记 | 调用对应本地工具/前端动作 |

### 4.3 RAG 证据规则

- `rag.results.length > 0` 不等于“可以展示证据”。
- 只有强相关结果才允许展示为“RAG 证据”。
- 弱相关结果最多作为内部参考，不能出现在证据卡里。
- 闲聊/身份问题禁止 RAG。
- 回答内容和证据卡必须一致：如果回答说没有依据，就不能展示证据卡。

### 4.4 模型输出安全规则

所有模型输出进入 UI 前必须经过清洗：
- 删除 `<think>...</think>`。
- 删除未闭合 `<think>` 后面的所有内容。
- 删除标题中的 Markdown、引号、句号。
- 标题只能来自用户首问或清洗后的简短主题。

### 5.1 已完成

- 桌面端基础框架。

- 学习空间聊天界面。
- 资料库上传与解析基础。
- 本地 RAG 检索。
- 真实 LLM 配置与 fallback。
- 知识图谱页面。
- 笔记页面。
- 设置页三组配置雏形。
- 目标/资料/图谱/笔记进入学习空间的上下文入口。

### 5.2 当前严重问题

| 问题 | 严重度 | 说明 |
|---|---:|---|
| 新建会话不干净 | P0 | 自动插入欢迎消息和策略按钮，违背自由会话预期 |
| `<think>` 泄漏 | P0 | 标题和侧栏可能显示模型思考内容 |
| RAG 误命中 | P0 | 闲聊问题可能命中无关资料 |
| Study 文件过大 | P1 | 单文件承载 UI、状态、RAG、会话、面板，维护风险高 |
| 数据流边界模糊 | P1 | Study 既像页面又像状态中心，容易互相污染 |
| 历史测试数据污染 | P1 | 本地 localStorage 里已有脏会话，需要迁移/清理策略 |
| 设计系统未收敛 | P2 | 多处卡片、按钮、面板状态不统一 |

## 6. MVP 成功标准

MVP 不是功能越多越好，而是满足以下可验证标准：

1. 新建自由会话为空白，只显示空状态和输入框。
2. 身份/闲聊问题不会触发 RAG。
3. RAG 证据卡只在强相关命中时出现。
4. 会话标题和侧栏标题永不显示 `<think>`。
5. 从资料/目标/图谱/笔记进入学习空间时，上下文准确可见。
6. “我懂了”只影响当前明确绑定的 task/node，不误改其他目标。
7. `pnpm tsc --noEmit` 和 `pnpm build` 必须通过。

## 7. 后续产品优先级

### P0：止血与可信度

- 干净新会话。
- 模型输出清洗。
- RAG 意图门控。
- 强/弱证据分离。
- 清理历史脏会话。

### P1：学习空间重构

- 拆分 Study 单文件。
- 明确会话状态模型。
- 明确上下文来源模型。
- 抽出面板系统。
- 抽出消息渲染系统。

### P2：学习闭环

- 当前任务推进。
- 节点掌握状态。
- 轻量测验验证。
- 学习记录报告。

### P3：个性化

- 学习画像增量更新。
- 薄弱点记录。
- 复习计划。
- 自动笔记整理。

---

## 8. 产品原则

1. **宁可少自动，也不要乱自动。**
2. **证据必须可信，弱相关不能冒充依据。**
3. **新会话必须干净，上下文必须显式。**
4. **Study 是学习执行区，不是所有模块的大杂烩。**
5. **先稳定闭环，再扩展 Agent。**
