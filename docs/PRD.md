## 1. 产品概述

### 1.1 一句话定义
**栖知是一个根据用户学习风格、节奏、情绪动态调整教学策略的 AI 私人导师桌面应用，让每个学习者都拥有一对一的专属老师。**

### 1.2 解决什么问题
传统教学是"一对多"：一个老师一种讲法，照顾不到全部人。
- 听不懂？不敢举手
- 进度跟不上？只能挂科
- 学习方式不适配？以为自己笨

**栖知用 AI 一对一的能力，把"一种讲法"变成"无数种讲法"，把"统一进度"变成"专属节奏"。**

### 1.3 产品定位
- ❌ 不是题库平台
- ❌ 不是网课平台
- ❌ 不是通用 ChatGPT
- ❌ 不是又一个 Web SaaS
- ✅ 是 **桌面端的 AI 私人学习陪伴系统**

### 1.4 形态
**macOS / Windows 桌面应用**（Tauri 打包）。
不做 Web 版本。理由：
- 学习是沉浸式的，桌面应用专注度更高
- 可以接入本地资料、本地通知、系统级快捷键
- 用户画像数据存本地，隐私更好
- 视觉表现力更强（毛玻璃、原生菜单、托盘）

## 2. 目标用户

### 2.1 首发主攻人群：大学生

**为什么是大学生**：
- 自主学习意愿强，不需要家长介入
- 没有 K12 的合规风险
- 颜值和体验敏感，适合打磨作品
- 反馈快、传播快、迭代容易
- 学习场景多样：专业课、考证、考研、技能学习

### 2.2 用户画像
| 维度 | 描述 |
|---|---|
| 年龄 | 18-25 岁 |
| 身份 | 在校大学生、研究生 |
| 痛点 | 老师讲得听不懂 / 自学没方向 / 学习效率低 / 容易摆烂 |
| 设备 | 笔记本电脑为主 |
| 付费力 | 弱（作品期不收费） |

### 2.3 典型场景
1. **期末复习**：上传课件 → AI 出题 → 针对性复习
2. **专业课跟不上**：学习空间里让 AI 用"换种讲法"重新讲一遍
3. **考研备考**：长期学习计划 + 每日追踪 + 遗忘曲线复习
4. **技能学习**：自定义目标（如学 Python） → AI 拆解路径 → 找资源 → 教学

## 3. 信息架构（重要！）

### 3.1 整体架构

```
栖知（桌面应用）
│
├─ 🚪 首次启动
│   └─ 学习风格测试（一次性 onboarding，5-8 分钟）
│       └─ 生成用户画像 → 存本地 → 后续后台静默更新
│
├─ 🏠 看板（Dashboard）              ← 全局总览
├─ 📚 学习空间（Study）⭐            ← 核心工作区（重头戏）
├─ 🎯 我的目标（Goals）              ← 计划全景
├─ 🗂 资料库（Library）               ← 上传资料 + AI 出题
├─ 🌳 知识图谱（Graph）               ← 完整知识网络
├─ 📝 笔记（Notes）                   ← AI 协助整理的笔记
└─ ⚙️ 设置（Settings）                ← 含「我的学习画像」可视化
```

### 3.2 关键认知（这次必须澄清）

#### ① 学习风格测试 ≠ 一个常驻模块
- 它是**首次启动的 onboarding 流程**，5-8 分钟
- 测完后结果**存本地** → 形成用户画像
- 后续在用户学习过程中**后台静默观察 + 自动更新**画像
- 用户也可以在「设置」里看自己的画像、手动重测

#### ② 学习不是一个按钮，是一整个工作空间
之前我把"AI 教学""番茄钟""找资源""临时笔记"想成分散的功能。
**正确的做法**：它们是「学习空间」这个核心工作区里的**协作面板**，一屏全部在线。

#### ③ 看板是总览，不是塞所有功能
看板只展示**汇总信息**：今日任务、节奏曲线、知识图谱缩略、复习提醒、成就。
真正"做事"在「学习空间」。

## 4. 核心功能（MVP）

### 4.1 🚪 学习风格测试（Onboarding）

**目的**：在用户首次启动时了解 ta 怎么学得进去，生成专属画像。

**测试组合**（5-8 分钟）：
- **VARK**（16 题）→ 视觉 / 听觉 / 阅读 / 动手 偏好
- **Felder-Silverman 简版**（12 题）→ 活跃-反思 / 感知-直觉 / 视觉-言语 / 序列-全局
- **Grit Scale 简版**（8 题）→ 毅力与坚持度

**输出**：
- 一张「学习风格画像」卡片（可视化雷达图）
- 一段 AI 生成的个性化解读
- 推荐的教学策略组合

**画像存储**：
- 本地 SQLite（用户隐私优先）
- 字段：vark / felder / grit / 派生策略偏好 / 最后更新时间

**画像自动更新机制**：
- 每次学习会话后，根据真实数据微调
  - 用户主动点"换种讲法" → 当前风格权重 -
  - 用户连续完成 → 当前难度权重 +
  - 用户跳过太多 → 节奏权重 -
- 每周生成一份「画像变化报告」，可在设置里查看

---

### 4.2 🏠 看板（Dashboard）

**定位**：全局总览，进入栖知第一眼看见的页面。

**模块**：
1. **顶部问候卡**：温和问候 + 连续学习天数 + 今日任务摘要 + 一句格言
2. **今日任务**：3-5 条 AI 拆解的当天任务，点击直接进学习空间
3. **学习节奏曲线**：30 天学习时长可视化
4. **知识图谱缩略**：知识点掌握度简图，点击进完整图谱
5. **本周成就**：3 个徽章
6. **复习提醒**：基于遗忘曲线的复习建议
7. **浮动「灵」入口**：随时唤起 AI 对话

详见 [DASHBOARD_DESIGN.md](../design/DASHBOARD_DESIGN.md)

---

### 4.3 📚 学习空间（Study）⭐ 核心王牌

**定位**：栖知的灵魂页面。**所有学习相关的能力一屏到位**。

**三栏布局**：

#### 左栏（260px）：当前学习计划树
- 树状展开当前目标 → 章节 → 知识点
- 高亮"正在学习"的节点
- 底部今日任务进度条

#### 中栏（弹性宽度，最大）：AI 教学对话区
- 与「灵」的实时对话
- 苏格拉底式提问 / 故事化讲解 / 类比 / 步骤拆解
- 每条灵的消息下方 4 个快捷按钮：
  - 🔄 换种讲法
  - ⬇️ 更深入
  - ✅ 我懂了
  - ⏭ 跳过
- 顶部可手动切换教学风格

#### 右栏（300px）：辅助面板（垂直堆叠）
- 🍅 **番茄钟**：圆形进度环 + 暂停/跳过
- 🌳 **当前位置**：迷你知识图谱，高亮当前知识点
- 📝 **临时笔记**：随时记录灵感
- 🔍 **AI 找到的资源**：自动联网搜索的相关视频/文章

**关键体验点**：
- 学一个知识点，**所有相关能力一屏完成**
- 右栏可折叠（专注模式：只留 AI 对话）
- 番茄钟与学习时长统一计入数据

---

### 4.4 🎯 我的目标（Goals）

**定位**：长目标的全景管理。

**功能**：
- 多个目标并行（如：高数上 / 考研英语 / Python 入门）
- 每个目标展开为：里程碑 → 周计划 → 任务
- 用 SVG 树状图展示
- 可手动调整、可让 AI 重新规划

---

### 4.5 🗂 资料库（Library）

**定位**：用户上传的学习资料 + AI 自动出题。

**功能**：
- 上传 PDF / Word / Markdown / 图片（OCR）
- AI 自动学习资料 → 提取知识点 → 用户确认
- 基于资料自动生成试卷（选择 / 填空 / 简答）
- 资料可关联到具体目标

**RAG 流程**：
```
上传 → 解析 → 切片 → 向量化 → 入本地知识库
   → AI 阅读理解 → 提取知识点 → 出题
```

---

### 4.6 🌳 知识图谱（Graph）

**定位**：可视化展示用户全部知识网络。

**功能**：
- 力导向图（D3）
- 节点颜色：绿（掌握）/ 橙（学习中）/ 红（缺漏）
- 节点大小：重要度
- 点击节点 → 详情面板（掌握度、学习历史、关联节点、"去学习"按钮）
- 可缩放、可拖拽、可按主题过滤

---

### 4.7 📝 笔记（Notes）

**定位**：学习过程中的笔记沉淀，AI 协助整理。

**功能**：
- 按主题分组
- Markdown 编辑
- AI 自动提取重点 / 易混淆点
- 支持公式（KaTeX）
- 导出为 PDF / Anki / Markdown

---

### 4.8 ⚙️ 设置 · 含「我的学习画像」

**定位**：用户偏好 + 学习画像可视化。

**模块**：
- **我的学习画像**（重点）
  - 雷达图 + 文字解读
  - 上次更新时间 + "重新测试"按钮
  - 一行小字："栖知会持续观察你的学习习惯，自动微调画像"
- **学习偏好**：每日目标时长 / 番茄时长 / 提醒时间
- **AI 教学偏好**：默认讲解风格
- **通知设置**
- **数据导出**
- **关于栖知**

## 7. 非功能性需求

### 7.1 性能
- 应用启动 < 2s
- AI 响应首 token < 1s
- 切页流畅 60fps
- 内存占用 < 300MB（Tauri 优势）

### 7.2 数据安全 & 隐私
- 用户画像、笔记、上传资料**默认存本地**
- 不上传服务器，不用于训练
- 支持一键导出全部数据
- 选择性云同步（用户可选）

## 9. 风险与对策

| 风险 | 对策 |
|---|---|
| AI 幻觉胡说八道 | 关键内容 RAG + 事实核查 + 来源标注 |
| 第一版功能太多做不完 | 严格 MVP，超出范围一律拒绝 |
| 桌面应用打包学习曲线 | 早期就用 Tauri 跑通 hello world，再做功能 |
| 看板/学习空间做得不够美 | 设计阶段不省时间，做到惊艳为止 |
| 学习画像不准 | 提供手动调整入口，不强行用算法判断 |

---

## 10. 不做什么（同样重要）

第一版**绝对不做**：
- ❌ Web 版本（专心做桌面）
- ❌ 社交功能
- ❌ 排行榜（容易劝退中等用户）
- ❌ 复杂的付费体系
- ❌ 多人协作
- ❌ 直播课
- ❌ K12 题库
- ❌ 强制每日打卡（栖知是温柔的）
