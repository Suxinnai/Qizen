## 1. 产品概述

### 1.1 一句话定义
**栖知是一个围绕用户自己的资料、知识图谱、学习记录与真实大模型能力构建的 AI 私人学习导师桌面应用。**

### 1.2 解决什么问题
传统学习产品常见问题：
- 讲法单一，不适配个人学习方式
- AI 会胡说，缺少对用户资料的真实依据
- 学习资料、知识点、练习、笔记各自割裂
- 学习过程有记录，但很少形成真正的个性化教学

**栖知要做的是：**
- 用用户自己的资料做 RAG 检索依据
- 用真实大模型生成更自然的教学回答
- 用知识图谱串联知识点关系
- 用学习记录和画像逐步走向自适应教学

### 1.3 产品定位
- ❌ 不是题库平台
- ❌ 不是网课平台
- ❌ 不是通用聊天机器人
- ✅ 是 **桌面端的 AI 学习工作台 + 私人导师系统**

### 1.4 当前形态
- 平台：**Windows / macOS 桌面应用**（Tauri）
- 当前阶段：**MVP 中后期 / 可用原型阶段**
- 当前重点：先把学习核心闭环做真，再逐步补产品层模块

## 2. 目标用户

### 2.1 首发主攻人群：大学生 / 研究生 / 自主学习者

**原因**：
- 自主学习需求强
- 场景复杂：专业课、考研、证书、技能学习
- 对资料管理、结构化学习、AI 辅助的需求明显
- 对桌面端工作流接受度高

### 2.2 典型使用场景
1. 上传课程资料 → AI 基于资料讲解与出题
2. 围绕知识图谱节点逐步推进学习
3. 从目标 / 任务进入学习空间完成当天学习
4. 在学习空间提问，优先从资料库检索依据
5. 将学习记录沉淀为后续个性化教学基础

## 3. 当前信息架构（以当前实现为准）

```text
栖知（桌面应用）
│
├─ 🚪 首启 · 学习风格测试（Onboarding）
├─ 🏠 看板（Dashboard）
├─ 📚 学习空间（Study）⭐ 核心工作区
├─ 🎯 我的目标（Goals）
├─ 🗂 资料库（Library）
├─ 🌳 知识图谱（Graph）
├─ 📝 笔记（Notes）
└─ ⚙️ 设置（Settings）
```

### 当前认知重点
- 学习风格测试是 **Onboarding 流程**，不是长期常驻模块
- 真正学习发生在 **学习空间 Study**
- 资料库和知识图谱现在都已经不是占位页，而是已有核心原型
- 当前最重要的产品价值是：
  **资料库 → RAG → 学习空间真实回答 → 证据展示 → 出题**

## 4. 当前已实现的核心功能

### 4.1 Onboarding 学习画像
- 学习风格测试已完成
- 可生成并保存学习画像
- 可展示主偏好 / 次偏好 / 推荐策略

### 4.2 Dashboard / Goals / Notes / Settings
- Dashboard 已有任务、节奏图、复习提醒基础展示
- Goals 已有目标 / milestone / task 展示
- Notes 已支持查看与编辑
- Settings 已支持：
  - 学习画像展示
  - 学习偏好设置
  - LLM provider 配置
  - 模型测试连接

### 4.3 Library 资料库
- 已支持上传并解析：
  - PDF
  - DOCX
  - Markdown / TXT
- 已支持：
  - 摘要提取
  - 重点提取
  - 正文预览
  - 练习生成基础结构
- 已支持资料与知识图谱节点自动关联
- 已支持带资料上下文进入学习空间

### 4.4 Graph 知识图谱
- 已有独立图谱页
- 支持节点查看、详情展示、相关节点
- 支持关联资料展示
- 支持带节点上下文进入学习空间

### 4.5 Study 学习空间
- 已有聊天式学习界面
- 已支持从资料库 / 图谱带上下文进入
- 已接入本地 RAG 检索
- 已支持证据展示：
  - 命中资料
  - 命中片段
  - 命中重点
  - 命中原因
- 已支持基于命中资料出题
- 已支持每道题证据回链

### 4.6 真实 LLM API 接入
- 已支持：
  - OpenAI-compatible
  - Anthropic Claude
- Study 当前链路：
  - 先本地 RAG
  - 再调用真实模型
  - 失败 fallback 到本地回答
- 已支持 Settings 中测试连接

### 4.7 学习记录埋点（MVP）
- 已记录：
  - 用户问题
  - 当前 resourceId / nodeId
  - 命中资料标题
  - 是否使用真实模型
  - 是否 fallback
  - 是否生成练习题

## 7. 成功标准（当前阶段）

MVP 中后期成功，不是“功能越多越好”，而是要做到：
- 用户能上传资料并得到真实解析结果
- 用户提问时，回答尽量建立在资料依据上
- 回答有证据可看
- 出题有证据可追溯
- 模型可配置、可测试、可 fallback
- 学习过程开始可追踪
