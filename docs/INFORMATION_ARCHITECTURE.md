# 栖知 信息架构（IA）

> 版本 v0.2 · 2026-04-20 · 作者：灵 & 沐灵
> 更新：已同步当前真实页面状态，Library / Graph / Study / Settings 不再按占位页描述

---

## 一、整体地图

```text
栖知（桌面应用）
│
├─ 🚪 [应用首启] 学习风格测试（Onboarding）
│   └─ 一次性流程，完成后进入看板
│
├─ 🏠 看板（Dashboard）
├─ 📚 学习空间（Study）⭐
├─ 🎯 我的目标（Goals）
├─ 🗂 资料库（Library）
├─ 🌳 知识图谱（Graph）
├─ 📝 笔记（Notes）
└─ ⚙️ 设置（Settings）
```

---

## 二、页面职责（当前版本）

| 页面 | 当前状态 | 核心职责 |
|---|---|---|
| Onboarding | 已完成基础版 | 生成学习画像 |
| Dashboard | 已完成基础版 | 汇总今日任务、学习节奏、复习提醒 |
| Study | 已完成核心原型 | 对话式学习、RAG 检索、证据展示、出题 |
| Goals | 已完成基础版 | 展示目标 / milestone / task |
| Library | 已完成核心原型 | 资料上传、解析、查看、联动学习空间 |
| Graph | 已完成核心原型 | 节点查看、关联资料、联动学习空间 |
| Notes | 已完成基础版 | 笔记查看与编辑 |
| Settings | 已完成增强版 | 偏好设置、LLM 配置、测试连接 |

## 三、关键页面流转

### 1. 首次使用
```text
应用启动
→ Onboarding
→ 生成学习画像
→ Dashboard
```

### 2. 日常学习主链路
```text
Dashboard / Goals / Library / Graph
→ Study
→ 提问
→ 本地 RAG 检索
→ 真实模型回答 / fallback
→ 证据展示
→ 基于命中资料出题
```

### 3. 资料驱动学习链路
```text
Library 上传资料
→ 解析
→ 自动关联节点
→ 带着资料进入 Study
→ 结合资料回答
```

### 4. 图谱驱动学习链路
```text
Graph 选择节点
→ 查看节点详情
→ 查看关联资料
→ 进入 Study 学这个节点
```

## 四、各页面入口与出口（以当前实现为准）

### 4.1 Onboarding
**入口**
- 首次启动自动进入
- Settings 中可重新测试

**出口**
- 完成后进入 Dashboard

---

### 4.2 Dashboard
**入口**
- 默认首页
- 侧栏进入

**出口**
- 今日任务 → Study
- 复习提醒 → Study
- 侧栏跳转到其他模块

> 当前 Dashboard 仍偏总览页，后续会逐步接入真实学习记录。

---

### 4.3 Study（核心工作区）
**入口**
- Dashboard 任务
- Goals 任务
- Library 资料详情
- Graph 节点详情

**当前能力**
- 聊天式学习
- 本地 RAG 检索
- 真实 LLM 回答
- fallback 本地回答
- 证据卡展示
- 练习生成
- 题目证据回链

**当前出口**
- 可返回任意侧栏页面
- 可从右侧 resource / graph 面板继续跳转理解当前内容

---

### 4.4 Goals
**入口**
- 侧栏

**出口**
- 当前主要是查看目标结构
- 后续应补：从 task 直接进入 Study / AI 重规划目标

---

### 4.5 Library
**入口**
- 侧栏
- 上传资料工作流

**当前能力**
- 上传并解析资料
- 查看摘要、重点、正文预览
- 查看关联知识节点
- 带着资料进入 Study

**出口**
- → Study（带 resourceId / nodeId）
- 后续应补：原文定位、搜索高亮、资料管理操作

---

### 4.6 Graph
**入口**
- 侧栏

**当前能力**
- 查看图谱节点
- 展示节点详情
- 展示相关节点
- 展示关联资料
- → Study（带 nodeId）

**出口**
- → Study
- 后续应补：掌握度驱动更新、图谱路径推荐

---

### 4.7 Notes
**入口**
- 侧栏
- 当前主要作为独立页使用

**当前能力**
- 查看笔记
- 编辑笔记
- 查看 AI 重点与易混淆点

---

### 4.8 Settings
**入口**
- 侧栏

**当前能力**
- 学习画像展示
- 番茄时长 / 教学风格 / 提醒开关
- LLM provider 配置
- 测试连接
- 查看学习交互次数

**后续应补**
- 更安全的密钥存储
- 数据管理
- 导出 / 清理

## 六、当前 IA 结论

栖知现在的架构已经从“页面占位”进入“核心学习闭环已成形”的阶段。

### 已经成形的主链路
- 资料 → 学习空间
- 图谱 → 学习空间
- 学习空间 → RAG → 证据 → 出题

### 下一阶段要补的，不是再堆页面，而是：
- 个人中心
- 学习报告
- 自适应推荐
- 长期记忆与进度系统

也就是说，当前 IA 的核心任务已经从“把页面列出来”转向“把产品层补完整”。
