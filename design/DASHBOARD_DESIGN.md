# 栖知 看板设计方向

> 版本 v0.3 · 2026-04-20
> 更新：已按当前真实实现同步，区分“已落地”和“后续设计目标”

---

## 一、文档定位

这份文档不再把看板写成纯概念稿，而是分成两层：
- **当前已落地版本**：代码里真实存在的 Dashboard
- **下一阶段设计目标**：后续继续迭代的方向

---

## 二、当前已落地版本

当前 `Dashboard` 已经是一个 **可用的总览页 MVP**，不是占位页。

### 已实现模块
1. **顶部问候卡**
   - 渐变背景
   - 学习画像摘要
   - 连续学习天数文案
   - 栖知格言
   - 叶子装饰

2. **今日任务卡**
   - 展示当天任务
   - 支持勾选完成
   - 本地持久化

3. **学习节奏卡**
   - 展示最近 30 天学习时长柱状节奏图
   - 数据来自本地 `studyStats`

4. **复习提醒卡**
   - 展示推荐复习项
   - 可直接进入 `Study`

### 当前页面角色
Dashboard 当前不是“完整数据分析中心”，而是：
- 用户进入应用后的第一屏
- 用来快速判断今天该做什么
- 帮用户从总览页进入 Study

---

## 三、当前设计问题

随着功能推进，Dashboard 目前还存在这些明显问题：

### 1. 数据还不够真实
- 学习节奏数据目前仍偏演示化
- 复习提醒逻辑仍比较轻
- 还没有真正接入 Study 的完整学习记录分析

### 2. 信息密度仍偏轻
当前只有 3 块核心卡片，产品感还不够完整。

### 3. 缺少核心学习闭环反馈
现在 Dashboard 还没有清晰展示：
- 最近提问次数
- 命中资料次数
- 真实模型调用情况
- fallback 比例
- 节点掌握变化

---

## 四、下一阶段设计目标

### P1：把 Dashboard 从总览页升级为“学习驾驶舱”

建议下一阶段补这些模块：

#### 1. 学习记录摘要
- 今日提问次数
- 命中资料次数
- 今日生成练习题数
- 是否主要依赖真实模型 / fallback

#### 2. 最近学习上下文
- 最近学习的资料
- 最近进入的图谱节点
- 最近一次学习的问题主题

#### 3. 节点掌握趋势
- 哪些节点正在提升
- 哪些节点最近答错较多
- 哪些节点值得复习

#### 4. 更可信的复习提醒
从固定卡片升级为基于学习记录和命中情况的推荐。

---

## 五、推荐布局（下一版）

```text
顶部：问候卡 + 今日状态摘要
中部第一行：今日任务 / 学习节奏
中部第二行：学习记录摘要 / 最近学习内容
底部：复习提醒 / 最近掌握变化
```

原则是：
- 不把 Dashboard 做成信息垃圾场
- 不抢 Study 的工作区角色
- 只做“总览 + 导航 + 反馈”

---

## 六、交互原则

### 1. 看板负责“告诉你去哪里”
- 今日任务 → 进 Study
- 复习提醒 → 进 Study
- 最近资料 → 进 Library / Study
- 最近节点 → 进 Graph / Study

### 2. 看板不负责“完成学习”
真正的学习动作仍然在 `Study` 中完成。

### 3. 看板要越来越依赖真实学习记录
后续所有关键卡片都应该逐步由：
- `studyRecord`
- `studyStats`
- `libraryItems`
- `knowledgeGraph`

这些真实数据驱动，而不是静态假数据。

---

## 七、当前结论

Dashboard 现在已经有了不错的视觉基础和总览结构，但还没真正成为学习系统的“驾驶舱”。

下一阶段不该再只是调配色或改排版，而是要把：
- 学习记录
- 资料命中
- 节点变化
- 模型调用情况

这些真实信号接进来。

这样看板才不是好看而已，才是真的有用。
