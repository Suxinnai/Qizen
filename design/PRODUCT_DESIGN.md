# 栖知产品设计文档

> 版本：v0.4-reset  
> 日期：2026-04-24  
> 目的：重新定义当前产品的交互、视觉与学习空间设计规则

---

## 1. 设计目标

栖知的设计目标不是“看起来像聊天软件”，而是创造一个清晰、可信、低干扰的学习工作台。

核心体验：
- 左侧：导航与学习会话管理。
- 中间：当前学习内容与对话。
- 右侧：可收起的学习工具面板。
- 底部：输入与附件入口。

---

## 2. 全局布局

```text
┌──────────────────────────────────────────────┐
│ 顶栏：应用标题 / 时间 / 主题 / 通知           │
├──────────────┬───────────────────────┬───────┤
│ 左侧栏        │ 主内容区                │ 右工具 │
│ 菜单/会话     │ 学习会话 / 空状态 / 问答 │ 面板栏 │
├──────────────┴───────────────────────┴───────┤
│ 输入区：附件 / 文本 / 语音 / 发送              │
└──────────────────────────────────────────────┘
```

### 2.1 左侧栏

左侧栏有两种模式：

| 模式 | 用途 |
|---|---|
| 主菜单 | Dashboard / Study / Goals / Library / Graph / Notes / Settings |
| 会话列表 | 最近学习会话、新建会话、切换历史 |

规则：
- 会话标题必须干净，不显示模型思考或 Markdown。
- 新建会话按钮只创建“空白学习空间”，不自动塞欢迎消息。
- 历史会话点击后恢复该会话自己的 messages/context。

### 2.2 中间学习区

中间区域只显示当前学习会话。

状态分三类：

| 状态 | UI |
|---|---|
| 空白自由会话 | 空状态 + 输入框，不显示助手消息 |
| 带上下文会话 | 简短上下文提示 + 可选右侧面板 |
| 已有对话 | 消息流 + 证据/练习卡 |

### 2.3 右侧工具栏

右侧默认显示竖向工具按钮：
- 番茄钟
- 资料/依据
- 笔记
- 知识图谱

规则：
- 工具栏不遮挡正文。
- 面板展开时靠右浮出，不改变主内容宽度过大。
- 空白自由会话不自动打开任何面板。
- 只有明确上下文或 AI trigger 才打开面板。

## 3. 学习空间交互设计

### 3.1 新建会话

用户点击左侧 “+”：
1. 清空 activeConversationId。
2. 清空 messages。
3. 清空 latestRag / practiceSet / practiceHint。
4. 清空 activePanel。
5. 显示空状态。

空状态文案：

> 新的学习会话  
> 这里是干净空间。直接输入问题，或从目标、资料库、笔记、知识图谱带上下文进来。

禁止行为：
- 禁止自动发送助手欢迎语。
- 禁止自动打开图谱面板。
- 禁止自动生成标题。
- 禁止自动绑定默认任务。

### 3.2 用户提问

用户输入后流程：

```text
用户输入
→ 清洗文本
→ 意图识别
→ 是否学习问题？
  ├─ 否：直接本地/模型回答，不触发 RAG
  └─ 是：执行 RAG 检索
       → 是否强相关？
          ├─ 是：回答 + 证据卡
          └─ 否：通用回答，不展示证据卡
→ 保存会话
→ 首轮用户问题后生成标题
```

### 3.3 RAG 证据卡

证据卡只在强相关时显示。

卡片内容：
- 资料标题
- 相关度标签
- 命中片段
- 命中重点
- 是否来自当前资料/当前节点

弱相关处理：
- 不显示证据卡。
- 可以作为内部 hint，但回答不能说“根据资料”。

### 3.4 消息渲染

消息进入 UI 前统一处理：
- 删除 `<think>` 与未闭合思考标签。
- 简单 Markdown 清洗或渲染。
- 标题级 Markdown 不能裸露成正文噪音。

## 4. 模块间数据流设计

### 4.1 资料库 → 学习空间

传入：
- `source = library`
- `resourceId`
- `nodeId?`

Study 行为：
- 显示当前资料摘要。
- 可默认打开资料面板。
- 回答优先当前资料。

### 4.2 知识图谱 → 学习空间

传入：
- `source = graph`
- `nodeId`

Study 行为：
- 展示当前节点。
- 可打开图谱面板。
- 检索时优先关联该节点资料。

### 4.3 目标 → 学习空间

传入：
- `source = goal`
- `taskId`

Study 行为：
- 当前学习目标绑定到任务。
- “我懂了”只推进该 task。
- 如果无明确 node，不自动打开图谱。

### 4.4 笔记 → 学习空间

传入：
- `source = note`
- `noteId`

Study 行为：
- 当前会话标题可来自笔记标题。
- 回答可参考笔记内容。
- 不默认展示 RAG 证据，除非另有资料命中。

## 5. 视觉规范补充

### 5.1 风格关键词

- 安静
- 克制
- 温和
- 可信
- 有学习感

### 5.2 空状态

空状态不能像错误页，也不能像聊天欢迎语。

推荐结构：
- 大标题：新的学习会话
- 一句话说明：这里是干净空间
- 可选快捷入口：从资料库进入 / 从目标进入 / 从笔记进入

### 5.3 按钮状态

| 类型 | 用途 |
|---|---|
| 主按钮 | 发送、上传、开始学习 |
| 次按钮 | 查看资料、生成练习 |
| 轻按钮 | 更深入、给我例子 |
| 危险按钮 | 清除画像、删除资料 |

### 5.4 证据状态

| 状态 | 颜色 | 展示 |
|---|---|---|
| 强相关 | 绿色 | 可显示证据卡 |
| 中相关 | 黄色 | 可提示“需确认”，默认不作为主要证据 |
| 弱相关 | 灰/红 | 不展示为证据 |

## 6. 组件拆分建议

当前 `Study.tsx` 过大，应拆为：

```text
routes/Study.tsx
components/study/StudyHeader.tsx
components/study/MessageList.tsx
components/study/MessageBubble.tsx
components/study/StudyInput.tsx
components/study/RightToolDock.tsx
components/study/PomodoroPanel.tsx
components/study/ResourcePanel.tsx
components/study/NotePanel.tsx
components/study/GraphPanel.tsx
lib/study/intent.ts
lib/study/session.ts
lib/study/rag-policy.ts
lib/study/message-sanitize.ts
```

拆分原则：
- 页面只负责布局。
- intent/rag/session 不写在 JSX 文件里。
- 面板互相独立。
- 存储层只做持久化，不做 UI 决策。

## 7. 当前设计结论

现在最大的问题不是界面不好看，而是**交互语义不稳定**：
- 新会话不是真的新。
- 弱相关资料被显示成证据。
- 面板自动弹出缺少规则。
- Study 承担了太多职责。

下一阶段设计重点：

**先把学习空间的状态机做对，再继续扩功能。**
